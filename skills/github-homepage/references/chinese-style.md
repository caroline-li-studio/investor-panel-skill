# Chinese README Style

Use natural Chinese open-source/product language. Avoid stiff translation.

## Tone

Prefer:

- "我做了一个..."
- "它解决的问题是..."
- "你会得到..."
- "最简单的方式..."
- "不要只复制..."
- "适合用来..."

Avoid:

- "本项目旨在..."
- "提供了一套完善的解决方案..."
- "赋能用户..."
- "通过先进技术实现..."

## Selling Point Patterns

```text
不是 X，而是 Y。
```

Example:

```text
不是一个 AI 分析师给答案，而是一场投资委员会辩论。
```

```text
真正的卖点不是 A，而是 B。
```

Example:

```text
真正的卖点不是“多一个结论”，而是看到不同视角为什么不同意。
```

```text
你把 X 交给它，它会 Y。
```

Example:

```text
你把一个 repo 交给它，它会先找卖点，再重写首页、主图、安装方式和示例。
```

## Section Names

Good Chinese section names:

- 一句话安装
- 你会得到什么
- 它适合做什么
- 怎么工作
- 示例提示词
- 边界
- 目录结构

## Codex Skill Install Copy

```text
请从这个仓库安装 Codex skills：
<repo-url>

请把 skills/ 下的完整技能文件夹安装到我的 Codex skills 目录中。
不要只复制 SKILL.md。
```

When shared resources are needed:

```text
请把 skills/ 下的完整技能文件夹安装到我的 Codex skills 目录中，包括 skills/_shared。
不要只复制 SKILL.md。
```
